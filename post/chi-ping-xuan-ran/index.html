CPU/GPU硬件区别

CPU(Central Processing Unit)

现代计算机整体系统的运算核心/控制核心

GPU(Graphics Processing Unit)

进行绘图运算工作的专用微处理器，是连接计算机和显示终端的纽带

  1. 渲染
  2. 应用计算能力：音视频编解码、视频滤镜、特效处理、人工智能机器学习

屏幕成像和卡顿问题

屏幕撕裂

产生撕裂的原因
  1. 帧缓存区上一帧的位图数据(帧缓存区)。
  2. 帧缓存完成计算后新的帧缓存区数据

垂直同步+双缓存区

  • 这是解决屏幕撕裂的方法，垂直同步信号保证缓存区的所有数据绘图完成之后，继续绘制下一个缓存区的数据。
  • 这个方案会引发另一个问题：掉帧

掉帧 Jank

产生掉帧的原因 (存疑，需补充)
  • GPU/CPU在渲染时耗时过长，苹果建议UI不建议图层过多。
  • 双缓存解决屏幕撕裂的问题，不能解决掉帧问题

三缓存区解决掉帧

这是一种策略，不能完全解决掉帧，尽量优化。苹果官方采用的是双缓存策略

屏幕卡顿的本质

图形、图像渲染流程

  1. CPU处理图片解码得到基本图元&纹理 - Application
  2. GPU 几何处理，图元装配(顶点着色器 & 偏远着色器) - Geometry Processing
  3. GPU 光栅化，找到图元位置把颜色绘制上去(此时数据还没有显示到屏幕上，而是在帧缓存存储器)- Rasterization
  4. GPU 通过显示控制器将上面光栅化的结果做相应的像素处理 - Pixel Processing

  • 顶点着色器 ：确定图形具体在硬件(屏幕)显示位置的转化
    • 动画中的平移、缩放、旋转
    • 在手机2D的屏幕上显示3D数据，顶点着色器的深度计算技术
    • 实例 - 抖音特效
  • 偏远着色器 ： 计算每一个像素点的颜色值

离屏渲染的本质

当有多个View图层叠加显示时，设备渲染该图层是需要时间去计算相应操作处理的（圆角剪切、颜色计算...），苹果不可能同事完成多个图层的这些操作，而是一个一个图层分别做这些相同的处理之后再一层一层合成为最终图层，此时系统会额外开辟一个临时缓冲区用来把每一步计算结果合成最终结果(即发生离屏渲染)，计算完成后放在帧缓存区完成绘制显示。

离屏渲染时发生在GPU的过程

触发离屏渲染的两个原因

  1. 图层过多，无法快速完成绘图，绘图需要消耗大量时间
  • 遮罩层(例：画一个绿色的树)
    1. 获取遮罩层Texture
    2. 获取绿色背景Texture
    3. 合成最终结果
  • 毛玻璃效果
    1. 获取渲染内容Texture
    2. 获取捕获内容Texture
    3. 垂直模糊
    4. 水平模糊
    5. 合成最终结果
  • 圆角
  1. ShouldRasterize光栅化
  • 如果layer不能被复用，则没有必要打开光栅化
  • 如果layer不是静态的，需要被频繁修改，比如处在动画之中，那么开启离屏渲染反而影响效率
  • 离屏渲染缓存内容有时间限制，缓存内容100ms内容如果没有被使用，那么他就会丢弃，无法再进行复用
  • 离屏渲染缓存空间有限，超过2.5倍屏幕像素大小的话，也会失控，且无法进行复用了

常见触发离屏渲染的几种情况

  1. 使用了mask的layer(layer.mask)
  2. 需要进行剪裁的layer(layer.maskToBounds / View.clipsToBounds)
  3. 设置了组透明度为yes，并且透明度不为1的layer(layer.allowsGroupOpacity / layer.opacity)
  4. 添加了投影的layer(layer.shadow)
  5. 采用了光栅化的layer(layer.shouldRasterize)
  6. 绘制了文字的layer(UILabel,CATextLayer,CoreText ...)

  6. 阴影 - shadowPath绘制
